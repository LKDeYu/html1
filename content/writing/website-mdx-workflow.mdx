## 为什么转向 MDX

这个网站是自用的，不是多人系统。对我来说，写文章最重要的是内容能稳定保存、方便迁移、容易用 Git 追踪，而不是一定要有后台表单。

MDX 的优势在于：文章就是文件，元信息写在 frontmatter 里，正文用 Markdown 写，需要时还能嵌入组件。

## 一篇文章怎么写

在 `content/writing` 目录新建一个 `.mdx` 文件，例如：

```mdx
---
title: "文章标题"
date: "2026-05-19"
summary: "一句话摘要。"
tags: ["学习记录", "项目"]
imageUrl: "/images/campus/study-1.jpg"
---

## 正文标题

这里写正文。
```

文件名会变成文章链接，比如 `website-mdx-workflow.mdx` 会对应 `/blog/website-mdx-workflow`。

## 适合我的地方

- 可以直接用编辑器写
- Git 能记录每次修改
- 不需要登录后台
- 搬家或重构时更容易迁移
- 项目、笔记和生活记录可以统一成同一种内容格式
